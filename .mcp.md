# Make `.mcp.json` file instead of this file

because of fnm you must use full paths for command and args (maybe not for node path)

- command is path to node executable
- args is path to your mcp server file (in this case src/first-ever-mcp.ts)

we named "demo-server" but you can name it whatever you want in src/first-ever-mcp.ts
we named our tool "add" but you can name it whatever you want in src/first-ever-mcp.ts

## `.mcp.json`

```json
{
  "mcpServers": {
    "demo-server": {
      "type": "stdio",
      "command": "/run/user/<>/fnm_multishells/<session_id>/bin/node",
      "args": [
        "/home/<user>/<path_to_project>/src/first-ever-mcp.ts"
      ],
      "env": {
        "NODE_OPTIONS": "--no-deprecation"
      }
    },
    "weather-server": {
      "type": "stdio",
      // works with just putting node also
      "command": "node",
      "args": [
        "<absolute path to the /src/weather.ts>"
      ],
      "env": {
        "NODE_OPTIONS": "--no-deprecation"
      }
    }
  }
}
```
